# 등기 배송상태 일괄조회 웹앱 - Vercel/Next.js 버전

엑셀/CSV 파일을 업로드하면 `등기번호`만 우체국 Open API에 조회하고, 내부용 `순번/생년월일/고객번호`는 브라우저 안에서만 매칭해 최종 CSV로 출력합니다. 이름 컬럼은 사용하지 않습니다.

## 핵심 기능

- 엑셀 `.xlsx/.xls` 및 `.csv` 업로드
- 순번/생년월일/등기번호/고객번호 컬럼 자동 인식 및 수동 변경
- 우체국 Open API 자동조회
- 작업 중 1건 조회 완료 때마다 브라우저 `localStorage` 자동 저장
- 브라우저가 꺼져도 최근 작업 복구
- 실패 건만 재조회, 전체 재조회
- 조회 완료 시 전체 결과 CSV 자동 다운로드
- 전체/성공/실패 CSV 다운로드
- 작업 백업 JSON 다운로드 및 불러오기
- 개인정보 최소화: 서버에는 등기번호만 전송, 순번/생년월일/고객번호는 서버로 보내지 않음

## Vercel 배포 방법

1. 이 폴더를 GitHub 저장소에 업로드합니다.
2. Vercel에서 해당 GitHub 저장소를 Import 합니다.
3. Vercel Project Settings → Environment Variables에 아래 값을 추가합니다.

```text
EPOST_SERVICE_KEY=공공데이터포털_인증키
```

4. Redeploy 합니다.

## API 키 발급

공공데이터포털(data.go.kr)에서 아래 API 중 하나를 활용신청합니다.

- 과학기술정보통신부 우정사업본부_국내우편물 종적 조회 서비스
- 과학기술정보통신부 우정사업본부_우체국 종적 조회

개발단계/운영단계 자동승인 API입니다. 실제 트래픽 한도는 공공데이터포털의 현재 조건을 확인하세요.

## 로컬 실행

```bash
npm install
cp .env.example .env.local
npm run dev
```

브라우저에서 `http://localhost:3000` 접속.

## 입력 파일 예시

```csv
순번,생년월일,등기번호,고객번호
1,900101,1234567890123,C001
2,850203,1234567890124,C002
```

## 주의

- 서버 API에는 `등기번호`만 전송합니다.
- 순번/생년월일/고객번호는 브라우저 상태와 localStorage에만 저장됩니다.
- 실제 회사 업무망에서 사용할 경우, Vercel 외부 배포보다 내부망 배포를 검토하세요.

## v0.1.1 수정사항
- 최초 접속 시 브라우저 localStorage에 작업 인덱스가 없으면 `JSON.parse(null)` 결과가 `null`이 되어 화면이 깨지는 문제를 수정했습니다.
- 저장된 작업 인덱스가 비정상 형식일 때도 빈 목록으로 안전하게 처리합니다.


## v0.1.2 수정사항
- 이름 컬럼을 제거하고, 입력/출력 기준 컬럼을 `순번/생년월일/등기번호/고객번호`로 정리했습니다.
- CSV 헤더와 실제 값 개수가 어긋나던 문제를 수정했습니다. `순번`에는 원본 순번 또는 자동 생성 순번만 출력됩니다.
- 고객번호 컬럼 자동 인식 후보를 고객번호 계열로 좁혔습니다.
- CSV 다운로드는 브라우저 다운로드 방식입니다. 사용자의 기본 다운로드 폴더 또는 브라우저가 묻는 저장 위치에 저장됩니다.

## v0.1.3 수정사항
- 조회가 정상 완료되면 전체 결과 CSV를 자동 생성하여 브라우저 다운로드를 시작합니다.
- 단, 브라우저 보안상 서버가 사용자 PC 특정 폴더에 몰래 저장하지는 못하며, 다운로드 폴더 또는 브라우저 설정에 따른 저장 위치로 내려받습니다.

## v0.1.4 수정사항
- 파일 업로드 후 등기번호 컬럼이 자동 인식되면 작업 생성과 동시에 자동조회가 바로 시작됩니다.
- 이전 작업과 60% 이상 일치하면 기존 결과를 병합하고 미조회/중단/실패 건만 자동으로 이어서 조회합니다.
- [이 설정으로 작업 생성] 또는 [이전 결과 병합 후 이어서 조회]를 누른 경우에도 자동조회가 바로 시작됩니다.
- 사용자가 별도로 [이어서 조회] 버튼을 누르지 않아도 됩니다.
- 입력/출력 기준 컬럼을 `순번/등기번호/고객번호`로 간소화했습니다. 서버에는 등기번호만 전송됩니다.


## v0.1.6 수정사항
- 업로드한 국내우편물 배송조회서비스 명세서 기준으로 API 호출을 정렬했습니다.
- 호출 대상은 `retrieveLongitudinalService/getLongitudinalDomesticList` 단일 엔드포인트로 정리했습니다. 통합 종적조회 fallback은 제거했습니다.
- 요청 파라미터명을 명세서 표기와 맞춰 `serviceKey` 소문자로 변경했습니다.
- 등기번호 검증을 숫자 13~15자리로 조정했습니다.
- 응답 헤더의 `successYN`, `returnCode`, `errMsg`를 진단 정보에 반영했습니다.
- `debug=1` 호출 시 실제 호출 URL(키 마스킹), HTTP 상태, 원문 일부, 파싱 결과를 함께 반환합니다.


## v0.1.8 변경사항

- 순번 컬럼을 엑셀에서 읽지 않고 업로드된 데이터 행 순서대로 1부터 자동 생성합니다.
- `고객번호`, `관리번호`처럼 `번호`가 들어간 컬럼이 순번으로 잘못 인식되는 문제를 방지했습니다.
- 기존 브라우저 저장 작업을 불러와도 화면/CSV 출력 시 순번이 1부터 다시 정렬됩니다.


## v0.1.8 변경사항

- 결과표 화면에 `배달일자` 컬럼을 추가했습니다.
- API 응답의 배달일자 필드명이 다른 경우를 대비해 `dlvyDe`, `dlvyDate`, `deliveryDate` 등 여러 후보를 인식합니다.
- 별도 배달일자 필드가 없더라도 종적 이력에서 `배달완료` 이벤트가 있으면 그 이벤트 날짜를 배달일자로 보정합니다.
- API 테스트 메시지에 배달일자 확인값을 같이 표시합니다.


## v0.1.9 변경사항
- 공공데이터포털 국내우편물 종적조회 문서 기준 필드 매핑 정정
- 배달일자: `dlvyDe`만 우선 사용
- 배달상태: `dlvySttus` 사용
- 종적목록 날짜: `dlvyDate`로 분리
- `dlvyDe`가 비어 있는 경우에만 `processSttus=배달완료` 이벤트의 `dlvyDate`로 보정
- `debug=1` 응답에 `rawFields` 추가
