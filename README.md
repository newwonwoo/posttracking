# 등기 배송상태 일괄조회 웹앱 - Vercel/Next.js 버전

## 현재 버전: v0.2.5

공공데이터포털 **국내우편물 종적조회 API** 문서의 응답값을 앱/CSV에 모두 반영했습니다.

### 반영한 API 응답값

| API 필드 | 앱/CSV 컬럼 | 설명 |
|---|---|---|
| `applcntNm` | 발송인 | API가 제공하는 발송인명 |
| `addrseNm` | 수취인 | API가 제공하는 수취인명 |
| `rgist` | 등기번호 | 조회 등기번호 |
| `pstmtrKnd` | 우편물종류 | 우편물 종류 |
| `trtmntSe` | 취급구분 | 취급 구분 |
| `dlvyDe` | API배달일자 | 문서상 배달일자 |
| `dlvySttus` | 배달상태 | 배달완료, 반송배달 등 대표 상태 |
| `dlvyDate` | 처리일자 / 종적이력 | 종적목록 각 이벤트 날짜 |
| `dlvyTime` | 처리시간 / 종적이력 | 종적목록 각 이벤트 시간 |
| `nowLc` | 처리우체국 / 종적이력 | 현재위치/처리 우체국 |
| `processSttus` | 처리현황 / 종적이력 | 배달준비, 배달완료, 반송 등 처리현황 |
| `detailDc` | 상세설명 / 미송달사유 / 종적이력 | 상세설명. 미송달 사유는 이 값을 우선 사용 |

### 미송달사유 산출 기준

- `배달완료/배송완료` 건은 미송달사유를 공란으로 둡니다.
- `반송`, `미배달`, `배달불능`, `부재`, `폐문`, `주소불명`, `보관기간경과` 등 미송달성 상태는 `detailDc`를 우선 미송달사유로 사용합니다.
- `detailDc`가 비어 있으면 `processSttus`를 보조값으로 사용합니다.

## 핵심 기능

- 엑셀 `.xlsx/.xls` 및 `.csv` 업로드
- 등기번호/고객번호 컬럼 자동 인식 및 수동 변경
- 순번은 엑셀 컬럼을 읽지 않고 업로드 행 순서대로 1부터 자동 생성
- 우체국 Open API 자동조회
- 작업 중 1건 조회 완료 때마다 브라우저 `localStorage` 자동 저장
- 브라우저가 꺼져도 최근 작업 복구
- 실패 건만 재조회, 전체 재조회
- 조회 완료 시 전체 결과 CSV 자동 다운로드
- 전체/성공/실패 CSV 다운로드
- 작업 백업 JSON 다운로드 및 불러오기
- 개인정보 최소화: 서버에는 등기번호만 전송, 고객번호는 브라우저 내부 매칭용

## Vercel 배포 방법

1. 이 폴더를 GitHub 저장소에 업로드합니다.
2. Vercel에서 해당 GitHub 저장소를 Import 합니다.
3. Vercel Project Settings → Environment Variables에 아래 값을 추가합니다.

```text
EPOST_SERVICE_KEY=공공데이터포털_인증키
```

4. Redeploy 합니다.

## API 키 발급

공공데이터포털(data.go.kr)에서 아래 API를 활용신청합니다.

- 과학기술정보통신부 우정사업본부_국내우편물 종적 조회 서비스

## 로컬 실행

```bash
npm install
cp .env.example .env.local
npm run dev
```

브라우저에서 `http://localhost:3000` 접속.

## 입력 파일 예시

```text

```

## 주의

- 서버 API에는 `등기번호`만 전송합니다.
- 고객번호는 브라우저 상태와 localStorage에만 저장됩니다.
- 실제 회사 업무망에서 사용할 경우, Vercel 외부 배포보다 내부망 배포를 검토하세요.
